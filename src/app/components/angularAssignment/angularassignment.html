Timeline & Requirements

Thank you for applying to Pezitr’s Angular Programming course.

We had many people apply for the class, so to be accepted you must finish a short assignment. The best submissions will be accepted into the course. Engineers that submit apps with extended functionality and improved UI are significantly more likely to be accepted into the class.

Application Timeline

  • Submit the assignment by 23:59, May 27, 2017. The assignment takes hours, and you will learn a lot, so please start early and submit as soon as you can.
  • We will admit up to 30 students into our class. We will notify you of the results between 1 and 2 weeks after your submissions.
  • The first class will take place on June 6. Classes will be on Mondays and Wednesday, 19:00-21:00.

Requirements

  • Be able to attend classes in Pune. Venue of the Classes will be shared soon.
  • Have some Object Oriented Programming experience.
  • Can dedicate 10-15+ hours per week on this 8-week course.
  • Experience in design can help you learn more quickly.
  • Be able to and be willing to learn a programming language quickly.
  • Interested in working on a final group project with a team during the course.
  • Be able to submit weekly homework assignments or project updates on time. If you fail to do that, you will not be eligible to continue hence you have to withdraw from the program.

Step 4: Deploy your application to Heroku

Throughout the course, we’ll be using Heroku to deploy our Rails applications. After you deploy to Heroku, you’ll be able to view your app at http://app-name.herokuapp.com.

  1. Go to www.heroku.com and create an account.
  2. Install the Heroku Toolbelt locally on your system.
  3. Add git add . all to your push.

  4. If you haven’t already initialized a git repository:

    1
2
    		 
    git init .
git commit -a -m "First commit of my super awesome app."
  5. Create a new app on Heroku with the command heroku create. It will give you a random name; you can change it later if desired.
  6. Push your code to Heroku and it will automatically restart the app for you: git push heroku master

Step 5: Submit the assignment

Once you’ve completed the application (including pushing to Heroku), please push your app via Github. All submissions occur via Github and we require each submission to follow a particular format which includes a clearly documented README with a GIF walkthrough and an issue raised as detailed below.

Submit your project by creating an issue on your repository and adding the Github handle (@pezitr) to the issue body so we can review the submission.

Also be sure to include a README.md containing a GIF walkthrough using LiceCap of the app demonstrating how it works with required user stories completed. Use this README template.

Note: All project submissions happen via Github. If needed, review how to push your apps to Github with this handy tutorial. Note that Cli automatically adds a .gitignore to your project as well to avoid commiting log files, temp files and other unnecessary files to git.
